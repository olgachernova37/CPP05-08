Реалізуй шаблон функції iter, який приймає 3 параметри та нічого не повертає:

Перший параметр — адреса масиву (вказівник на перший елемент).

Другий параметр — довжина масиву, передана як константне значення (const або за значенням).

Третій параметр — функція, яка буде викликана для кожного елемента масиву.

Твій шаблон функції iter повинен працювати з будь-яким типом масиву. Третій параметр може бути інстанційованим шаблоном функції.

Функція, яка передається як третій параметр, може приймати свій аргумент як за константним посиланням (const reference), так і за неконстантним посиланням (non-const reference), залежно від контексту.

💡 Подумай: Як підтримати роботу як з константними, так і з неконстантними елементами у твоїй функції iter?

📚 Теорія, яка знадобиться для захисту
1. Передача функції як параметра (Покажчик на функцію)
У C++ ми можемо передавати функції як аргументи інших функцій. Наприклад, якщо у нас є функція void print(int x), ми можемо передати її в iter. У шаблоні ми можемо просто описати цей параметр як універсальний тип F або детально розписати його як покажчик на функцію: void (*f)(T&) чи void (*f)(const T&).

2. Проблема const та non-const (Головний підвох завдання)
Якщо твій масив константний (наприклад, const int arr[]), то елементи цього масиву мають тип const int.
Якщо функція iter прийматиме лише покажчик на функцію виду void (*f)(T&) (без const), то компілятор видасть помилку, оскільки не можна передати const int у функцію, яка очікує звичайний int& (це б дозволило випадково змінити константні дані).

Як це вирішити?
Найпростіший, найнадійніший та найбільш елегантний спосіб у C++ — зробити третій параметр повністю шаблонним (наприклад, typename F). Тоді компілятор сам виведе правильний тип функції (чи вона приймає const T&, чи T&, чи взагалі передається за значенням T).

Напишемо декілька різнопланових тестів: для звичайних чисел, для константного масиву рядків, а також протестуємо функцію, яка модифікує значення (потребує non-const посилання).

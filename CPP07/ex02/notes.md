Вправа 02: Array (Масив)
Каталог для здачі: ex02/

Файли для здачі: Makefile, main.cpp, Array.{h, hpp} (та необов'язковий Array.tpp)

Заборонено: Немає

Розроби шаблон класу Array, який містить елементи типу T та реалізує таку поведінку і функції:

Конструктор за замовчуванням (без параметрів): Створює порожній масив (розмір 0, вказівник вказує на NULL).

Конструктор з параметром unsigned int n: Створює масив із n елементів, ініціалізованих за замовчуванням.

Підказка: Спробуй скомпілювати int * a = new int(); та виведи *a. (Це ініціалізує значення нулем. Тобі потрібно викликати new T[n](), щоб усі елементи пройшли ініціалізацію за замовчуванням).

Конструктор копіювання та оператор присвоювання: В обох випадках копіювання має бути глибоким (deep copy). Зміна оригінального масиву або його копії після копіювання не повинна жодним чином впливати на інший масив.

Виділення пам'яті: Ти мусиш використовувати operator new[] для виділення пам'яті. Превентивне виділення пам'яті (резервування "наперед") заборонено. Твоя програма ніколи не повинна звертатися до невиділеної пам'яті.

Доступ до елементів через оператор [ ] (subscript operator):

Якщо індекс виходить за межі масиву (менше 0 або більше/дорівнює розміру), має викидатися виняток типу std::exception (наприклад, std::out_of_range).

Примітка: Тобі потрібно реалізувати дві версії оператора []: одну для звичайних об'єктів, іншу — константну (const) для роботи з константними масивами.

Метод size(): Повертає кількість елементів у масиві (тип unsigned int). Ця функція не приймає параметрів і не має права модифікувати об'єкт (має бути const).

Здай файл main.cpp зі своїми тестами для перевірки всіх аспектів роботи класу.

📚 Теорія для захисту
1. Чому шаблон класу пишется в одному файлі?
Як і з шаблонами функцій, компілятор повинен бачити всю реалізацію методів класу Array у момент ініціалізації (наприклад, Array<int>). Тому весь код класу (і оголошення, і реалізацію методів) ми пишемо в одному файлі Array.hpp (або виносимо реалізацію в Array.tpp і підключаємо його в кінці Array.hpp). Ми напишемо все в Array.hpp для простоти та надійності.

2. Конструктор за замовчуванням для типу T
Коли ми пишемо new T[n]() (із круглими дужками в кінці), C++ гарантує, що для примітивних типів (як int, float, double) виділена пам'ять заповниться нулями (Value initialization). Без дужок там сміття.

3. Глибоке копіювання (Deep Copy)
Коли ми копіюємо один масив в інший, ми не можемо просто скопіювати вказівник. Якщо ми це зробимо, обидва об'єкти вказуватимуть на одну й ту саму область пам'яті, що призведе до double-free crash під час знищення об'єктів. Ми маємо виділити нову пам'ять через new[] та покласти туди копії кожного елемента.

4. Константність оператора []
Нам потрібні два перевантаження оператора []:

T& operator[](unsigned int index); — дозволяє читати та змінювати елементи: arr[0] = 5;.

const T& operator[](unsigned int index) const; — дозволяє лише читати елементи, якщо сам об'єкт масиву є константним (const Array<int> arr).
